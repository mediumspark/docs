QuickDraft supports Markdown natively. You can write using standard Markdown syntax and instantly preview how it renders — ideal for composing blog posts, README files, or structured notes.

## Writing Markdown

Type Markdown syntax directly into the editor just as you would in any text file. QuickDraft recognises standard CommonMark syntax and renders it correctly in the preview panel.

Here are the most common patterns:

````markdown
# Heading 1
## Heading 2

**bold text** and *italic text*

> This is a blockquote

- Item one
- Item two

1. First step
2. Second step

[Link text](https://example.com)

`inline code`

```
code block
```
````

You don't need to switch any mode to write Markdown — simply type your syntax and open the preview when you're ready to see the result.

## Previewing Markdown

Click the **+ Markdown** button below the editor to open the Markdown Preview panel. The panel renders your Markdown as formatted HTML in real time — headings, bold text, lists, links, blockquotes, and code blocks all display exactly as they would on a published page.

The preview updates as you type, so you can keep it open alongside your draft. When you're done, click **Close** to dismiss the panel and return to editing.

## Switching to code view

Click **\</> Code View** (also labelled **HTML Source**) below the editor to see the raw HTML generated from your content. This is useful when you need to copy the output and paste it into a CMS, email template, or web page.

Use the **Copy HTML** button in that panel to copy the entire HTML source to your clipboard. For a full breakdown of all available view modes, see the [View Modes guide](/guides/view-modes).

<Note>
  The toolbar and Markdown syntax work together. When you click a toolbar button — for example, **Bold** — QuickDraft inserts the corresponding Markdown syntax (`**like this**`) at your cursor. You can freely mix toolbar actions and hand-written Markdown in the same document.
</Note>
