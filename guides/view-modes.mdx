QuickDraft lets you switch between multiple views of your draft. Whether you need plain text to paste elsewhere, a formatted Markdown preview, or raw HTML, each view is one click away.

## Raw Text

Raw Text mode strips all formatting and shows your content as a plain string of characters — no Markdown syntax, no HTML tags.

- Click the **Raw Text** (📄) button below the editor to open the Raw Text panel.
- Click **Copy All** to copy the entire plain-text content to your clipboard in one action.
- Close the panel to return to editing at any time.

**When to use it:** Raw Text is ideal when pasting your draft into an app that doesn't support rich text — such as a plain-text email client, a terminal, or a form field — because it guarantees no unwanted formatting characters carry over.

## Markdown Preview

Markdown Preview renders your Markdown syntax as formatted HTML so you can see exactly how your document will look when published.

- Click the **+ Markdown** button below the editor to open the Markdown Preview panel.
- Headings, bold and italic text, lists, links, blockquotes, and code blocks all render correctly.
- The panel is **read-only** — to make changes, close the preview and edit in the main editor, then reopen the preview to check your updates.

For a guide to writing Markdown syntax in QuickDraft, see the [Markdown guide](/guides/markdown).

## HTML / Code View

Code View exposes the raw HTML that QuickDraft generates from your content.

- Click **\</> Code View** (also labelled **HTML Source**) below the editor to open the code panel.
- Click **Copy HTML** to copy the complete HTML source to your clipboard.
- Close the panel to return to editing.

**When to use it:** Code View is useful for developers who need to embed content in a website, CMS, or email template and want the exact HTML output without any manual conversion.

<Tip>
  Use **Raw Text** mode when you need to email a draft as plain text — it gives you clean, paste-ready content with no markup. Use **Markdown Preview** when sharing a draft with developers or content teams who need to see the rendered structure before it goes live.
</Tip>
